# Strategies Library

## Назначение

Здесь хранятся формализованные стратегии.

Стратегия отличается от идеи тем, что ее можно отдать разработчику или ИИ-инженеру для написания бэктеста без повторного объяснения.

## Когда идея становится стратегией

Идея становится стратегией, когда описаны:

- рыночная гипотеза;
- инструменты;
- входные данные;
- формула/логика сигнала;
- вход;
- выход;
- риск;
- ограничения;
- комиссии;
- проскальзывание;
- задержки;
- условия, где стратегия должна работать;
- условия, где стратегия может ломаться;
- план бэктеста.

## Формат файлов

```text
STRAT-YYYYMMDD-001-short-name.md
```

Использовать шаблон:

```text
STRATEGY_TEMPLATE.md
```

## Статусы стратегии

```text
draft               черновик
ready_for_backtest  готово для бэктеста
backtested          протестировано
needs_rework        нужно переработать
paper_candidate     можно тестировать в paper
rejected            закрыто
```

## Важное правило

Нельзя записывать стратегию как рабочую, если она не проверена на:

- комиссиях;
- проскальзывании;
- задержках;
- реалистичном исполнении;
- разных рыночных режимах.
